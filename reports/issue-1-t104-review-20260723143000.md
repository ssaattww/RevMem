## 初回finding

### P1: repository-wide Globalのmemory不整合

別contextを保存するとdisk上のGlobalは最新値へ進む一方、すでにmemoryへ読み込まれた他contextの`getCurrent`は旧Globalを保持していた。

対応:

- repository storage rootごとに最新Globalを保持するpublic facadeを追加した
- context stateは個別に維持し、Globalだけを全contextへ同期する回帰テストを追加した

### P1: T102 atomic transaction contractへの未接続

初回実装はcomplete context/Globalを保存できたが、T102で追加された`expected`完全snapshotの比較とstale拒否を実装していなかった。

対応:

- T102 transactionが構造的に代入可能な`ReviewStateTransactionLike`を追加した
- disk上のcontext/Globalとexpectedをdeep compareする`commit`を追加した
- stale時の`StaleReviewStateError`、commit failure notification、disk・memory不変テストを追加した
- 複数windowのread/write間競合を閉じるexclusive lockはT604へ残した

### P1: T102 merge後のbranch/PR test入口不整合

T104 branchへ`main`のpackage test行をそのままコピーすると、PR mergeでは成功してもT102 test sourceが存在しないT104 branch単体のpush runが失敗する構成だった。

対応:

- T104テストを既存`core-contracts.test.ts`からimportした
- T104 branchのpackageを共通祖先と同じ内容へ戻した
- branch単体はT104テスト、PR mergeはT102とT104の両テストを実行する構成にした
- PR mergeabilityを確認した

## 最終確認

### Routing

- Gitとpull requestは同じrepository hash rootを利用する
- workspaceは`storageUri`直下を利用する
- history、snapshot、cache、lock pathはrouteから一意に取得できる
- workspace routeにcache directoryを誤って設定していない

### Atomic persistence

- immutable context/Global documentの作成後にmanifestだけをcommit pointとして置換する
- manifest replacement失敗時は直前manifestが残り、orphan documentは現在状態から参照されない
- workspaceは単一documentのtemp-write、sync、renameで更新する
- write失敗後にmemory stateを更新しない

### Validation

- current schema version以外を拒否する
- repository ID、context ID、context kind不一致を拒否する
- manifest referenceのroot escapeを拒否する
- missing fileとJSON parse failureをload errorとして扱う

### Transaction CAS

- expectedとdiskのcontext/Globalが両方一致する場合だけnextを保存する
- 片方でも不一致ならstaleとして拒否する
- stale時はmanifest、context、Global、memoryを変更しない
- commit成功時だけrepository-wide Global memoryを更新する

### Tests・CI

- 初期Red run `29980087663`とfailure artifact `8552860223`を確認した
- 移行中のBuild failure、unit fixture failureを診断artifactから修正した
- review follow-upの回帰テストを追加した
- 最終run `29981865050`でInstall、Build、Lint、Unit、temporary Git integration、mock GitHub integration、VS Code Extension Hostが成功した
- workflowは失敗時だけ診断artifactを作成するため、最終runではartifact uploadがskipされた

## 結論

T104の終了条件を満たしている。保存先はrepository種別で分離され、Git/PRはmanifest-last、非Gitはworkspace-state単一documentで更新される。保存失敗時は直前状態とmemoryを維持し、成功後の再読み込み結果は一致する。T102のcomplete snapshot transactionはstale検出付きで接続され、後続storeは同じrouting contractを再利用できる。レビュー結果はpassとする。
