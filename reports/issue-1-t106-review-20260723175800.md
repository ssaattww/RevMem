## レビューで検出・修正した事項

### 1. ContextとGlobalの部分重複

初期実装では、Context区間の一部だけがGlobalでも有効な場合、Context区間全体のhoverへ`Global: active`を表示する可能性があった。

対応:

- 部分重複fixtureを追加
- Context区間をGlobal active / inactiveへ分割
- Global-only区間はContext区間を減算して重複描画を防止

結果:

- Actions run `29992192701`: success

### 2. テーマ対応の公開契約

初期実装はlight/dark別の固定RGBAであり、high contrastとユーザーテーマによる色上書きをmanifest契約として公開していなかった。

対応:

- `contributes.colors`へ背景色とOverview Ruler色を追加
- light、dark、high contrast、high contrast lightの既定値を定義
- VS Code側では`ThemeColor`を使用
- Extension Host testでmanifestを検証

結果:

- Red run `29992211357`: Extension Host test failure
- 診断artifact `ci-failure-diagnostics-29992211357-1`
- Green run `29992274740`: success

### 3. Global不整合によるContext装飾の消失

Contextの確実性判定がGlobal repository IDへ依存していたため、Global状態だけが不整合な場合もContext装飾が消えていた。

対応:

- Global不整合fixtureを追加
- ContextとGlobalの確実性判定をレイヤー単位に分離
- Global不整合時も確実なContext区間を維持

結果:

- Red run `29992884316`: unit test 1 failure
- 診断artifact `ci-failure-diagnostics-29992884316-1`
- Green run `29992936703`: success

## 最終確認

### 装飾モデル

- Context、Globalの各レイヤーを個別に検証する
- revision、path、line count、content hash、interval境界の不整合を非表示化する
- ContextをGlobalより優先する
- overlapを重複なしの区間へ分割する
- 未確認または不確実な範囲は空の装飾として返す

### controller

- visible editorだけを対象にする
- diff editorを空装飾へ置換する
- per-editor generationで古い結果を拒否する
- 設定変更時にDecoration Typeを再作成する
- コマンド成功後はデバウンスせずrefreshする
- 読込失敗時は古い確認済み表示を残さない

### VS Code adapter

- 背景はテーマ対応の半透明グレー
- ガターは既定有効
- Overview Rulerは既定無効
- hoverはContext、状態更新日時、Global状態を表示
- Markdown hoverはcommand非信頼、HTML無効
- ExtensionContext subscriptionでcontrollerとcommand registrationを破棄する
