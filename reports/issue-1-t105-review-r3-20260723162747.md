## リスク

- 未解決のリスクまたは後続対応:
  - 同一line countのままdocument内容がcommand実行中に変化するeditor version/content raceはT201までheld。現実装はcommand開始時snapshotのhash/line countを使うため、同時編集を避ける通常経路は成功する。
  - 状態commit成功後のhistory要求失敗はpartial successになり得る。公開契約に明記済みで、履歴本実装はT206以降の責務としてheld。
  - Markdown word checkはrepositoryにlint/cspell設定がないため`unsupported`。T105 tracking上の完了条件外であり、設定変更やユーザー判断は要求しない。
  - `npm run test:vscode`と`npm run package`によりignoredの`.vscode-test` cacheと`review-range-tracker-0.0.1.vsix`が生成された。git-visibleなsource変更はなく、reviewerの可視変更は本レポートだけ。
